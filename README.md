# pdf2md_mineru

A batch **PDF → Markdown** tool built on [MinerU](https://github.com/opendatalab/MinerU), great for turning papers and documents into Obsidian-ready notes in bulk.

> 中文说明见文末 [中文文档](#中文文档).

## What it does

- Scans every PDF in `input_pdfs/` (subfolders supported)
- Calls the MinerU CLI to convert them one by one
- Writes each PDF to its own subdirectory `output_md/<name>/`, so images and assets never collide
- Skips already-converted PDFs by default; failures don't stop the batch, and are written to logs and a summary report

## Project structure

```
pdf2md_mineru/
├── input_pdfs/          # Put PDFs to convert here
├── output_md/           # Conversion results (one subfolder per PDF)
├── logs/
│   ├── success.log
│   └── failed.log
├── scripts/
│   ├── batch_pdf_to_md.py
│   ├── batch_pdf_to_md_api.py
│   └── check_env.py
├── conversion_report.md   # Generated after a run
├── README.md
└── requirements.txt
```

## Requirements

- **Python 3.10 – 3.13** (MinerU's official recommendation)
- **MinerU CLI** installed (the `mineru` command is available)

### Check your environment

```powershell
cd pdf2md_mineru
python scripts/check_env.py
```

### Install MinerU (uv recommended)

```powershell
pip install --upgrade pip
pip install uv
uv pip install -U "mineru[all]"
```

Or with pip:

```powershell
pip install -U "mineru[all]"
```

The first run may download models, so keep your network available. Users in mainland China can set:

```powershell
$env:MINERU_MODEL_SOURCE = "modelscope"
```

## Adding PDFs

Put PDFs in `input_pdfs/`. Subfolders are supported, e.g.:

```
input_pdfs/
├── paper-a.pdf
└── books/
    └── chapter-1.pdf
```

Original files in `input_pdfs/` are **never modified or deleted**.

## Running the batch

Run from the project root `pdf2md_mineru/`:

### Safe test (process only 1 PDF)

```powershell
python scripts/batch_pdf_to_md.py --limit 1
```

### Full batch (skips already-converted by default)

```powershell
python scripts/batch_pdf_to_md.py
```

### CPU / lightweight mode (pipeline backend)

Without an NVIDIA GPU, use the `pipeline` backend (the script default). To set it explicitly:

```powershell
python scripts/batch_pdf_to_md.py --backend pipeline
```

For scanned PDFs you can boost OCR (extra args are passed through to mineru):

```powershell
python scripts/batch_pdf_to_md.py --limit 1 --mineru-extra -m ocr -l ch
```

### Overwrite existing results

```powershell
python scripts/batch_pdf_to_md.py --overwrite
```

## Converting with the MinerU API

If you don't want to download models locally or use local CPU/GPU, you can use MinerU's Precision Extract API. This requires an API token from the MinerU website and uploads your PDFs to the MinerU cloud for processing.

First set the token in the current PowerShell session:

```powershell
$env:MINERU_API_TOKEN = "your MinerU API token"
```

Safe test (process only 1 PDF):

```powershell
python scripts/batch_pdf_to_md_api.py --limit 1
```

Full batch (skips already-converted by default):

```powershell
python scripts/batch_pdf_to_md_api.py
```

Common API options:

| Option | Description | Default |
| --- | --- | --- |
| `--model-version` | MinerU API model, `pipeline` or `vlm` | `vlm` |
| `--language` | Document language, e.g. `en`, `ch` | `en` |
| `--ocr` | Enable OCR | off |
| `--batch-size N` | Files per upload batch, max 200 | `20` |
| `--timeout N` | Max seconds to wait per batch | `3600` |
| `--page-ranges` | Process only the given page ranges | none |

The API path generates `conversion_report_api.md`. The result zip returned by MinerU is downloaded and extracted into `output_md/<name>/`.

### Common options

| Option | Description | Default |
| --- | --- | --- |
| `--input` | Input PDF directory | `input_pdfs` |
| `--output` | Output directory | `output_md` |
| `--backend` | MinerU backend | `pipeline` |
| `--recursive` / `--no-recursive` | Recurse into subfolders | recurse |
| `--overwrite` | Overwrite existing Markdown | off |
| `--limit N` | Process at most N PDFs | unlimited |
| `--mineru-extra` | Extra args passed to `mineru` | none |

## Output

Each PDF maps to:

```
output_md/<pdf-name-without-extension>/
├── *.md              # Main Markdown (path recorded in conversion_report.md)
├── images/ etc.      # Assets generated by MinerU (actual layout may vary)
└── ...
```

MinerU may generate the `.md` in a subdirectory (e.g. `auto/`). The script **searches recursively** for all `.md` files under the output directory and records the actual path in `conversion_report.md`.

## Logs and reports

- Success: `logs/success.log`
- Failure: `logs/failed.log` (with error summaries)
- Summary: `conversion_report.md` (success / skip / fail counts plus per-file detail)

## FAQ

### 1. The `mineru` command isn't found

1. Confirm it's installed: `uv pip install -U "mineru[all]"` or `pip install -U "mineru[all]"`
2. Run `python scripts/check_env.py` to inspect your PATH
3. On Windows, if it's installed under your user dir, add the Python Scripts dir to PATH, e.g.
   `%APPDATA%\Python\Python311\Scripts`
4. Open a new terminal and try `mineru --version` again

### 2. Paths with spaces

The scripts use `pathlib` and list-form `subprocess`, so **no manual quoting is needed**. If your project path contains spaces, just `cd` into the project directory and run.

### 3. Large PDFs / slow conversion

- Test a single file first with `--limit 1`
- Use the `pipeline` backend (default)
- Tune environment variables (see the [MinerU CLI docs](https://opendatalab.github.io/MinerU/usage/cli_tools/)), e.g. `MINERU_PDF_RENDER_THREADS`
- For huge files, consider MinerU's `-s` / `-e` paging args:
  `python scripts/batch_pdf_to_md.py --mineru-extra -s 0 -e 49`

### 4. Poor recognition on scanned PDFs

- Use OCR mode: `--mineru-extra -m ocr`
- Specify the language: `-l ch` (Chinese), `-l en` (English), etc.
- If scan quality is poor, preprocess the PDF first (higher DPI, denoise)

### 5. Image paths in the output Markdown (Obsidian)

MinerU's `.md` usually references images with **relative paths** to the same or a sub directory. Recommended approach:

1. **Keep the whole folder**: copy or symlink the entire `output_md/<name>/` folder into your Obsidian vault, e.g. `Vault/PDF Notes/<name>/`
2. Open the main `.md` in that folder in Obsidian and images display correctly
3. If you want "one PDF, one note", you may keep only the main `.md` in the vault, but don't move the `.md` away from its `images/` and asset folders
4. Check the **actual Markdown path** recorded in `conversion_report.md` and open that file directly in Obsidian

```
YourVault/
└── Literature/              # or "PDF Notes"
    ├── paper-a/
    │   ├── paper-a.md       # actual path recorded in conversion_report
    │   └── images/
    └── books__chapter-1/    # PDFs from subfolders may carry a prefix
        └── ...
```

Option 1: **symlink** (saves disk)

```powershell
mklink /D "D:\Obsidian\YourVault\Literature\paper-a" "C:\path\to\pdf2md_mineru\output_md\paper-a"
```

Option 2: **copy** the relevant subfolder under `output_md` into your vault.

Option 3: set the Obsidian vault at the parent of `output_md` and browse via folder notes (good for large bulk imports).

## 常见问题

### 1. `mineru` 命令找不到怎么办？

1. 确认已安装：`uv pip install -U "mineru[all]"` 或 `pip install -U "mineru[all]"`
2. 运行 `python scripts/check_env.py` 查看 PATH
3. Windows：若装在用户目录，把 Python Scripts 目录加入 PATH，例如  
   `%APPDATA%\Python\Python311\Scripts`
4. 新开一个终端后再试 `mineru --version`

### 2. Windows 路径有空格怎么办？

脚本使用 `pathlib` 与 `subprocess` 列表传参，**无需手动加引号**。项目路径若含空格，只要正常 `cd` 到项目目录再运行即可。

### 3. PDF 很大、转换很慢怎么办？

- 先用 `--limit 1` 单篇试跑
- 使用 `pipeline` 后端（默认）
- 可调环境变量（见 [MinerU CLI 文档](https://opendatalab.github.io/MinerU/usage/cli_tools/)），例如 `MINERU_PDF_RENDER_THREADS`
- 超大文件可考虑 MinerU 的 `-s` / `-e` 分页参数：  
  `python scripts/batch_pdf_to_md.py --mineru-extra -s 0 -e 49`

### 4. 扫描版 PDF 识别效果不好怎么办？

- 使用 OCR 模式：`--mineru-extra -m ocr`
- 指定语言：`-l ch`（中文）、`-l en`（英文）等
- 扫描质量差时，可先对 PDF 做预处理（提高 DPI、去噪）

### 5. 输出 Markdown 里图片路径怎么处理（Obsidian）？

MinerU 生成的 `.md` 通常使用**相对路径**引用同目录或子目录下的图片。推荐做法：

1. **整包放入库**：把 `output_md/<文档名>/` 整个文件夹复制或软链接到 Obsidian vault，例如 `Vault/PDF笔记/<文档名>/`
2. 在 Obsidian 中打开该文件夹下的主 `.md` 即可正常显示图片
3. 若希望「一篇 PDF 一篇笔记」，可在 vault 里只保留主 `.md`，但不要单独移动 `.md` 而丢下 `images/` 等资源目录
4. 查看 `conversion_report.md` 中记录的**实际 Markdown 路径**，在 Obsidian 中直接打开该文件
